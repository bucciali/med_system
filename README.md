# Task Service

Сервис для управления задачами с HTTP API на Go.

## Требования

- Go `1.23+`
- Docker и Docker Compose


## Task Tracker
В существующий модуль трекера задач добавлена поддержка периодических шаблонов задач и генерации задач на конкретную дату.

Поддерживаемые типы периодичности:

-Ежедневные: создание задачи каждый n-й день.
-Ежемесячные: создание задачи в указанные дни месяца (1..30).
-На конкретные даты: создание задачи только в заранее заданные даты.
-Четные/нечетные дни месяца: создание только по четным или только по нечетным дням.

Идея решения
- Template: правило периодичности (что и когда нужно создавать).
- Task: конкретная задача на конкретную дату.


Для генерации реализован сценарий:
запрос на генерацию для даты;
получение всех шаблонов;
проверка, должен ли шаблон сработать на эту дату;
создание задачи с защитой от дублей (CreateIfNotExists).
API (добавлено)
- POST /api/v1/templates — создать шаблон периодической задачи.
- POST /api/v1/tasks/generate/{date} — сгенерировать задачи на дату (YYYY-MM-DD).
- GET /api/v1/tasks?date=YYYY-MM-DD — получить задачи на дату.

Остальные существующие CRUD-методы задач сохранены.


## Быстрый запуск через Docker Compose

```bash
docker compose up --build
```

После запуска сервис будет доступен по адресу `http://localhost:8080`.

Если `postgres` уже запускался ранее со старой схемой, пересоздай volume:

```bash
docker compose down -v
docker compose up --build
```

Причина в том, что SQL-файл из `migrations/0001_create_tasks.up.sql` монтируется в `docker-entrypoint-initdb.d` и применяется только при инициализации пустого data volume.

## API

Базовый префикс API:

```text
/api/v1
```

Основные маршруты:

- `POST /api/v1/tasks`
- `GET /api/v1/tasks`
- `GET /api/v1/tasks/{id}`
- `PUT /api/v1/tasks/{id}`
- `DELETE /api/v1/tasks/{id}`
